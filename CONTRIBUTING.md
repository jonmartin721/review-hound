# Contributing to Review Hound

Thanks for your interest in contributing! This project welcomes contributions of all kinds.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies with `uv sync`
4. Create a branch for your changes

## Development Setup

```bash
# Install dependencies
uv sync

# Run tests
pytest

# Run the development server
python -m reviewhound.cli web --reload
```

## Making Changes

- Keep changes focused and atomic
- Write tests for new functionality
- Ensure existing tests pass before submitting
- Follow the existing code style

## Submitting a Pull Request

1. Push your changes to your fork
2. Open a pull request against `main`
3. Describe what your changes do and why
4. Link any related issues

## Reporting Issues

When reporting bugs, please include:

- Steps to reproduce
- Expected vs actual behavior
- Python version and OS
- Relevant error messages or logs

## Code of Conduct

Be respectful and constructive. We're all here to learn and build something useful.

## Questions?

Open an issue if you have questions about contributing.
